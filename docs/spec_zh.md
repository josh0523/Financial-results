Requirements
1. Data Acquisition (資料來源)
請使用 requests 或 pandas 自動抓取以下官方數據（若無法直接爬取 HTML，請嘗試抓取官方提供的 CSV 連結）：
上市 (TSE)：證交所「公布注意有價證券資訊」(https://www.twse.com.tw/zh/announcement/notice.html)
上櫃 (OTC)：櫃買中心「公布注意有價證券資訊」(https://www.tpex.org.tw/zh-tw/announce/market/attention.html)
備註：請務必處理「民國年轉西元年」的日期格式問題 (e.g., 114/12/30 -> 2025-12-30)。

2. Data Parsing & Cleaning (資料清洗)
需要整合上市與上櫃的資料，統一欄位名稱：[市場別, 股票代號, 股票名稱, 日期, 注意交易資訊]
關鍵欄位解析：需使用 Regular Expression (Regex) 從「注意交易資訊」欄位中提取以下數值：
- 成交量放大倍數 (關鍵字：放大 x 倍 / 為 x 倍 / 之 x 倍)
- 漲幅 (關鍵字：漲幅達 x% / 漲幅 x%)
- TSE第一款 (關鍵字：第一款 / 累積收盤價漲幅)

3. Analysis Logic (核心篩選邏輯)
請計算每檔股票在最近 N 天(預設抓取近10日資料)的累積注意次數，並針對最新交易日的個股執行以下篩選：
- TSE個股：近六個交易日達到三次注意(不含第十款)，或是昨日公布第一到第三款的注意。
- OTC個股：近六個交易日達到三次注意。

4. User Interaction (使用者互動功能)
程式執行分析後，請設計一個 input() 介面。
提示語：「請輸入今日已公布自結的股票代號（用空白分隔）：」
程式需接收使用者輸入的代號字串 (e.g., "2330 2454 3008")。

5. Output Display (輸出呈現)
請使用 tabulate 或美觀的格式印出最終表格。
表格需包含欄位：| 市場 | 代號 | 名稱 | 風險評級 | 觸發原因 | 最後注意日 | 狀態 |
狀態欄位邏輯：
若該股代號在使用者輸入的清單中，顯示 ✅ 已公告 (排除)。
若無，顯示 ⚠️ 未公告 (高風險)。
